# Context Usage Guidelines

## Overview

When generating responses, it is important to report on how context was utilized to ensure transparency and clarity. This document serves as a guide on how to effectively report context usage.

## Constraints
- Always include a "Context Usage" section in your responses.
- Always report your context usage in a percentage format.
- When context usage reaches 80% or higher, advise the user to save all files and restart the session to avoid context overload.

---
_Adhering to these guidelines ensures clarity, reduces errors, and fosters effective communication within the team._
